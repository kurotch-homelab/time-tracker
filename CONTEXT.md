# Time Tracker

Time Tracker は、組織のクライアント・案件別に稼働時間と経費を記録し、集計・請求連携するための業務コンテキストです。

## Language

**稼働レコード**:
ユーザーが案件に対して行った作業時間を表す請求・集計単位。
_Avoid_: タイムログ, 作業ログ

**案件**:
クライアント配下で稼働・予算・請求設定をまとめる管理単位。
_Avoid_: プロジェクトなし, 案件未設定

**タスク**:
案件配下で稼働をさらに分ける任意の集計単位。
_Avoid_: チケット, ガントタスク

**未分類案件**:
参照先案件を失った稼働レコードを一時的に保持する、組織ごとの請求対象外システム案件。
_Avoid_: 案件なし, 孤児レコード

**記録範囲**:
案件に稼働レコードを追加できるユーザーの範囲。
_Avoid_: 閲覧範囲, 可視性

**閲覧権限**:
他ユーザーの稼働レコード詳細を読めるロール上の許可。
_Avoid_: 記録範囲, 案件公開

**時間帯**:
稼働レコードが占有する開始時刻から終了時刻までの範囲。
_Avoid_: 所要時間のみ, 日次合計

**アクティブタイマー**:
ユーザーごとに最大1つだけ存在する、停止前の進行中計測。
_Avoid_: 実行中レコード, 進行中ログ

**同期競合**:
オフライン中の変更がオンライン復帰時に同一ユーザーの一意性または重複禁止条件を満たせない状態。
_Avoid_: エラー, マージ待ち

**LWW**:
同期競合において `updatedAt` が最も新しい変更を正式データとして採用する方針。
_Avoid_: 手動マージ, サーバ優先

**送信履歴**:
freee へ下書きとして渡した請求明細の送信事実と送信時点の内容を保持する監査記録。
_Avoid_: 請求書, freee 請求書

**明細スナップショット**:
送信後の元データ編集に影響されない、freee 送信時点の明細内容。
_Avoid_: 明細参照, 現在値

**Webセッション**:
ブラウザのログイン状態を表す、サーバが発行し管理する認証状態。
_Avoid_: APIトークン, OIDCトークン

**外部連携トークン**:
ネイティブアプリまたは公開APIクライアントがAPIアクセスに使う発行済み認証情報。
_Avoid_: Webセッション, パスワード

**タイムシート**:
ユーザーの一定期間の稼働レコードを提出・承認するためのまとまり。
_Avoid_: 給与勤怠, 請求書

**ロック期間**:
締め日または承認後に稼働レコードの編集を制限する期間。
_Avoid_: 削除, アーカイブ

**Time Off**:
記録率・リマインダー・稼働率集計から除外する休暇または非稼働予定。
_Avoid_: 有給残数, 給与計算

**労務コスト**:
収益性分析で使う内部原価レート。
_Avoid_: 請求単価, 売上レート

**Timeline候補**:
本人の端末活動から作られる、稼働レコード化前のプライベートな記録候補。
_Avoid_: 監視ログ, 自動確定レコード

## Relationships

- A **User** has zero or one **アクティブタイマー**.
- A **稼働レコード** belongs to exactly one **案件**.
- A **稼働レコード** may belong to one **タスク**.
- An **Organization** has exactly one **未分類案件**.
- A **未分類案件** is an **案件** excluded from billing until reassignment.
- A **案件** has one **記録範囲**.
- **閲覧権限** is not granted by **記録範囲**.
- A **稼働レコード** has exactly one **時間帯**.
- A **同期競合** is resolved by **LWW** for MVP.
- A **送信履歴** has one or more **明細スナップショット**.
- A **明細スナップショット** may reference one or more **稼働レコード** or **経費**.
- A **User** may have multiple **Webセッション**.
- An **外部連携トークン** is not used as a browser **Webセッション**.
- A **タイムシート** contains one or more **稼働レコード** for one **User** and one period.
- A **ロック期間** prevents ordinary edits to **稼働レコード**.
- **労務コスト** is distinct from billable rates.
- A **Timeline候補** becomes a **稼働レコード** only after user confirmation.

## Example dialogue

> **Dev:** "PC とスマホがオフラインで別々にタイマーを開始した場合、どちらを残しますか?"
> **Domain expert:** "MVP では **LWW** です。`updatedAt` が新しい方だけを正式な **アクティブタイマー** または **稼働レコード** として採用します。"

> **Dev:** "freee 送信後に元の **稼働レコード** が編集されたら、送信済み金額も変わりますか?"
> **Domain expert:** "変わりません。**送信履歴** は **明細スナップショット** を持ち、送信時点の内容を再現できます。"

> **Dev:** "静的フロントからログインする時、ブラウザはAPIトークンを持ちますか?"
> **Domain expert:** "持ちません。Web は **Webセッション** を使い、**外部連携トークン** はネイティブアプリや公開API向けに分けます。"

> **Dev:** "30分だけ入力した **稼働レコード** は、重複チェックできますか?"
> **Domain expert:** "できます。保存時には必ず **時間帯** に正規化します。所要時間だけの正式データは作りません。"

> **Dev:** "オフライン中に記録した先の **案件** が削除済みだったら、**稼働レコード** は案件なしになりますか?"
> **Domain expert:** "なりません。組織の **未分類案件** へ自動で再割当し、請求対象外の要修正として扱います。"

> **Dev:** "公開案件に記録できるメンバーは、同じ案件の他人のメモも読めますか?"
> **Domain expert:** "読みません。**記録範囲** は記録可否だけを決め、他人の詳細は **閲覧権限** を持つ管理者と担当マネージャだけが読めます。"

> **Dev:** "Toggl Track parity の **タスク** はチケット管理ですか?"
> **Domain expert:** "違います。**タスク** は **案件** 内の稼働集計軸です。チケットやガントの進捗管理は外部ツールに任せます。"

> **Dev:** "**Timeline候補** は管理者が監視できますか?"
> **Domain expert:** "できません。本人だけが確認し、採用したものだけが正式な **稼働レコード** になります。"

## Flagged ambiguities

- オフライン競合は手動解決か自動解決かが曖昧だった。解決: MVP では **LWW** で自動採用し、採用されなかった変更は正式データにしない。
- freee 送信履歴が元レコード参照だけか送信時点の値も持つかが曖昧だった。解決: **送信履歴** は **明細スナップショット** を不変に保持する。
- 静的フロント構成で Web が API トークンを直接持つのかが曖昧だった。解決: Web は **Webセッション**、ネイティブ / 公開 API は後続フェーズの **外部連携トークン** として分離する。
- 所要時間だけの手動入力が重複禁止と矛盾していた。解決: **稼働レコード** は必ず **時間帯** を持ち、所要時間入力は保存前に正規化する。
- 案件必須と孤児レコードの「未分類」扱いが矛盾していた。解決: 組織ごとの **未分類案件** を用意し、孤児化した **稼働レコード** を自動再割当する。
- 案件の公開が記録可否なのか閲覧可否なのか曖昧だった。解決: **記録範囲** と **閲覧権限** を分離し、メンバーは他人の詳細 **稼働レコード** を閲覧できない。
- Toggl Track parity の追加で承認・ロック・収益性・Timelineが入るため、MVP仕様との境界を明確化した。解決: **タイムシート** / **ロック期間** / **労務コスト** / **Timeline候補** は Phase 8 / 9 の任意拡張として扱う。
